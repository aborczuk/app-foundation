# Behavior Map: ClickUp Control Plane

This file is the living "where does behavior live?" map for the clickup_control_plane package.
It is implementation-oriented and complements (not replaces) spec diagrams.

## Scope

- Runtime mode: FastAPI service (`uvicorn clickup_control_plane.app:app`)
- Endpoints: `GET /control-plane/health`, `POST /control-plane/clickup/webhook`, `POST /control-plane/workflow/completion`

## How To Read

- `Entrypoint`: first function/module that owns this behavior.
- `Config`: user-facing knobs that change behavior.
- `Observe`: logs/events that confirm behavior is active.
- `Tests`: fastest file(s) to inspect when changing behavior.

## Behavior Catalog

| Behavior | Entrypoint | Core Code Path | Config | Observe | Tests |
|---|---|---|---|---|---|

| Health endpoint | `src/clickup_control_plane/app.py::health` | `GET /control-plane/health` → `{"status": "ok"}` | none | HTTP 200 `{"status": "ok"}` | `tests/integration/clickup_control_plane/test_webhook_to_dispatch_flow.py` |
| Webhook HMAC signature verification | `src/clickup_control_plane/webhook_auth.py::assert_valid_clickup_signature` | Computes HMAC-SHA256 of raw request body with `CLICKUP_WEBHOOK_SECRET`; compares to `X-Signature` header (constant-time); raises `SignatureVerificationError` on mismatch | env: `CLICKUP_WEBHOOK_SECRET` | `401 invalid_signature` on failure | `tests/unit/clickup_control_plane/test_webhook_auth.py` |
| Dispatch policy evaluation | `src/clickup_control_plane/policy.py::evaluate_dispatch_policy` | (1) Allowlist scope check: task must be in an allowlisted `space_id` or `list_id`; (2) routing metadata extraction: `workflow_type`, `context_ref`, `execution_policy` from payload (top-level or nested); (3) `action_scope` validation against workflow type | env: `CONTROL_PLANE_ALLOWLIST` | Decision in `OrchestrationResult`: `dispatch`, `reject_scope`, `reject_missing_metadata`, `action_scope_violation` | `tests/unit/clickup_control_plane/test_policy.py` |
| Event deduplication + active-run lock | `src/clickup_control_plane/state_store.py::StateStore.record_event_and_acquire_lock` | Atomic SQLite transaction: (1) dedupe by `event_id` → `skip_duplicate` or `stale_event` if seen; (2) one-workflow-at-a-time by `task_id` → `reject_active_run` if lock held | env: `CONTROL_PLANE_DB_PATH` | Lock decision in `LockAcquisitionResult`: `dispatch`, `skip_duplicate`, `stale_event`, `reject_active_run` | `tests/unit/clickup_control_plane/test_state_store.py` |
| n8n dispatch | `src/clickup_control_plane/dispatcher.py::N8NDispatchClient.dispatch` | POST to `{N8N_DISPATCH_BASE_URL}/{route}` with structured payload; explicit timeout + cooperative cancellation via `asyncio.Event`; route map: `build_spec → /control-plane/build-spec`, `qa_loop → /control-plane/qa-loop` | env: `N8N_DISPATCH_BASE_URL`, `CONTROL_PLANE_REQUEST_TIMEOUT_SECONDS` | `dispatch_failed` decision on `DispatchTimeoutError`, `DispatchRequestError`, `DispatchRejectedError` | `tests/unit/clickup_control_plane/test_dispatcher.py` |
| Startup reconciliation (stale lock release) | `src/clickup_control_plane/reconcile.py::ReconciliationService.reconcile_stale_active_runs` | For each locally-`running` task run, probes ClickUp API; releases lock if ClickUp says task is terminal; keeps lock on probe failure (fail-safe) | env: `CLICKUP_API_TOKEN`, `CONTROL_PLANE_REQUEST_TIMEOUT_SECONDS` | `ReconciliationResult`: `confirmed_active`, `released_stale`, `probe_failures`, `release_failures` in `app.state` | `tests/integration/clickup_control_plane/test_webhook_to_dispatch_flow.py` |
| Pre-dispatch reconciliation checkpoint | `src/clickup_control_plane/reconcile.py::ReconciliationService.enforce_pre_dispatch_checkpoint` | Runs reconciliation before each dispatch; raises `ReconciliationCheckpointError` if any probe or release failures remain (unresolved drift) | same as startup reconciliation | `dispatch_failed` with `reason_code: reconciliation_checkpoint_failed` | `tests/integration/clickup_control_plane/test_webhook_to_dispatch_flow.py` |
| ClickUp outcome write-back | `src/clickup_control_plane/clickup_client.py::ClickUpOutcomeClient.write_decision_outcome` | Writes dispatch decision back to ClickUp task custom field; detects schema mismatch (`ClickUpSchemaMismatchError`) and returns `schema_mismatch` decision | env: `CLICKUP_API_TOKEN` | `schema_mismatch` decision if ClickUp custom field schema doesn't match expected; silent on `ClickUpError` (non-fatal) | `tests/unit/clickup_control_plane/test_clickup_client.py` |
| Dispatch orchestration (full flow) | `src/clickup_control_plane/service.py::DispatchOrchestrationService.process_event` | Coordinates: policy → lock → dispatch → outcome write-back; returns terminal `OrchestrationResult` with `decision` and `reason_code` | all env vars (via dependencies) | `WebhookAcceptedResponse` HTTP 202 with `event_id` and `decision` on success; `502 dispatch_failed` on n8n failure | `tests/integration/clickup_control_plane/test_webhook_to_dispatch_flow.py` |

| Completion callback endpoint | `src/clickup_control_plane/app.py::workflow_completion` | `POST /control-plane/workflow/completion`; validates `X-Completion-Token`; routes by `status`: `waiting_input` → pause flow, `timed_out` → blocked flow, terminal → lock release + outcome write | env: `CONTROL_PLANE_COMPLETION_TOKEN` | HTTP 202 with decision envelope; `401 invalid_completion_token` on bad token | `tests/contract/test_clickup_control_plane_contract.py` |
| HITL pause flow (`waiting_input`) | `src/clickup_control_plane/app.py::workflow_completion` → `state_store.upsert_paused_run` | Sets ClickUp task to `CONTROL_PLANE_HITL_WAITING_STATUS`; upserts `PausedTaskRun` in local DB with `human_input_request` context; requires structured `human_input_request` in payload | env: `CONTROL_PLANE_HITL_WAITING_STATUS` (default: `Waiting for Input`) | `waiting_input` decision in response; paused-run record in `paused_task_runs` table | `tests/integration/clickup_control_plane/test_webhook_to_dispatch_flow.py` |
| HITL resume via operator comment | `src/clickup_control_plane/service.py::DispatchOrchestrationService._resume_paused_run` | Detects `HITL_RESPONSE:` prefix on ClickUp comment (case-insensitive) or `human_input_response` field; dedupe-inserts event; resolves route from paused run; dispatches resume request with `resume_run_id`, `human_input_response`, `human_input_prompt`; clears paused run on success | none | `input_resumed` decision; dispatch includes HITL context fields | `tests/integration/clickup_control_plane/test_webhook_to_dispatch_flow.py` |
| HITL timeout → blocked | `src/clickup_control_plane/app.py::workflow_completion` (timed_out branch) | Sets ClickUp task to `CONTROL_PLANE_HITL_BLOCKED_STATUS`; releases active run lock; clears paused run; writes `timed_out` lifecycle outcome | env: `CONTROL_PLANE_HITL_BLOCKED_STATUS` (default: `Blocked`), `CONTROL_PLANE_HITL_TIMEOUT_SECONDS` (default: `86400`) | `timed_out` decision; task set to blocked status in ClickUp; no orphan state in local DB | `tests/integration/clickup_control_plane/test_webhook_to_dispatch_flow.py` |
| n8n cancel dispatch | `src/clickup_control_plane/dispatcher.py::N8NDispatchClient.cancel_run` | Best-effort POST to `{N8N_DISPATCH_BASE_URL}/control-plane/cancel-run` with `task_id`, `run_id`, `event_id`, `reason`, `occurred_at_utc`; raises `DispatchRejectedError` on non-2xx (caller handles, does not block state cleanup) | env: `N8N_DISPATCH_BASE_URL` | Cancel payload received by n8n; `DispatchRejectedError` on failure (best-effort, non-blocking) | `tests/unit/clickup_control_plane/test_dispatcher.py` |
| Manual cancel (controlled→non-controlled) | `src/clickup_control_plane/service.py::DispatchOrchestrationService._cancel_active_run_from_manual_status_change` | Detects when status transitions from a controlled status to a non-controlled status while an active run exists; sends best-effort cancel signal; unconditionally releases active lock + clears paused run; persists `cancelled_by_operator` terminal decision | none | `cancelled_by_operator` decision; active lock released + paused run cleared regardless of cancel signal success | `tests/integration/clickup_control_plane/test_webhook_to_dispatch_flow.py` |
| Lifecycle completion summaries | `src/clickup_control_plane/app.py::_build_completion_outcome` | Builds operator-visible outcome payload from completion status: `completed` → severity `info`; `failed`/`timed_out` → severity `error`; other → severity `warning`; includes `run_id`, artifact links, HITL prompt context | none | `TaskOutcomePayload` written to ClickUp via `ClickUpOutcomeClient` | `tests/contract/test_clickup_control_plane_contract.py` |
| Paused-run persistence | `src/clickup_control_plane/state_store.py::StateStore.upsert_paused_run / get_paused_run / clear_paused_run` | `paused_task_runs` table; ON CONFLICT upsert for idempotency; `clear_paused_run` uses `BEGIN IMMEDIATE` for atomicity; optional `run_id` match on clear | env: `CONTROL_PLANE_DB_PATH` | Paused-run record visible in `paused_task_runs`; cleared after resume/cancel/timeout | `tests/unit/clickup_control_plane/test_state_store.py` |
